General lab instructions to help labs run smoothly

  • Read through the lab instructions before coming to the lab.

Upon completion of this lab, you should be able to understand the linked list implementation of the Binary Tree ADT and implement some binary tree operations.

Exercise 1: Understanding the BinaryTreeNode class

Download the file BinaryTreeNode.java. Look at the picture of the BinaryTreeNode in Topic 14 page 45 of the lecture notes, and examine the code to see how the node is implemented.
  • Change the privacy modifiers on the attributes so that they are private rather than protected.
  • Add getter and setter methods for all the attributes.
  • Make sure your new BinaryTreeNode.java compiles.

Exercise 2

  • Download the files LinkedBinaryTree.java, as well as the files ArrayList.java and ArrayUnorderedList.java provided here. Download as well the interfaces you need from the code webpage, as well required exception classes, iterators, ...
  • In LinkedBinaryTree.java, change all usage of the attributes of the BinaryTreeNode class (such as root.left) in the provided code, to invocations of the getter and setter methods you wrote in Exercise 1.
  • Complete the following methods in LinkedBinaryTree.java:
    • getRoot, isEmpty, size
    • contains. Hint: use findAgain.
    • iteratorPreOrder, preorder, iteratorPostOrder, and postorder. Hint: Model your code for these on the methods iteratorInOrder and inorder. The main thing that you need to change is where the node is visited in preorder and postorder.
  • Complete the method toString. Hint: printing the data elements of the tree is basically just a traversal of the tree. Choose one of the traversal orders (preorder, inorder, or postorder) and use exactly the same idea of using a temporary unordered list as in the provided method iteratorInOrder: create a new unordered list, call the method preorder, or inorder, or postorder, which will add the elements of the tree to the unordered list, call the toString method on the unordered list.
  • For now, you do not need to complete iteratorLevelOrder. However, you will need to return something from within the body of this method in order to have the code compile: just have it return null.
  • To test your implementation, download the file TestBinaryTree.java. You will also need to insert an additional constructor into your LinkedBinaryTree.java; it is in the file LBTConstructor.txt, and is used to build the sample tree in TestBinaryTree.java. Check the comments in the code for TestBinaryTree.java to see what the constructed tree looks like.
