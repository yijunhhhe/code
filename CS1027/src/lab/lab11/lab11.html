Exercise 1 - Traversing a Binary Search Tree in Descending Order

Recall that an inorder traversal of a Binary Tree does the (recursive) traversal like this: traverse left subtree, visit root, traverse right subtree. For a Binary Search Tree, this results in the tree items being visited in ascending order. To visit the items in descending order, we switch the left and right traversals in the recursive algorithm.

Recall that the LinkedBinaryTree class has the method iteratorInOrder. Let's call our method that does the right,root,left variation of inorder traversal iteratorInOrderDescending. There are many ways we could add this method; for example, it could be added to LinkedBinaryTree.java. However, since it is most relevant to a Binary Search Tree, we could also add it to our LinkedBinarySearchTree (which extends LinkedBinaryTree). Or, we could add it to BinarySearchTreeList (which extends LinkedBinarySearchTree). We will call this class IterableBinarySearchTreeList, and you will implement the following two methods (only) in that class:

  • public Iterator iteratorInOrderDescending(), which returns an iterator which iterates over all elements in the tree in descending order. Hint: see the iteratorInOrder method in LinkedBinaryTree. You can copy its code to this method, and change it to use your inorderDescending method.
  • private inorderDescending(BinaryTreeNode node, ArrayUnorderedList tempList), which performs a recursive inorder traversal in descending order. Hint: this is nearly identical to the inorder method in LinkedBinaryTree. Here, instead of going left to right, we want to go right to left.

Exercise 2 - Creating a Balanced Binary Search Tree List

Download the files BuildBSTList.java and iteratorLevelOrder.txt and put the code of the iteratorLevelOrder method into your LinkedBinaryTree.java from Lab 10 (this is the only method you did not complete in Lab 10). Note that it requires the ArrayUnorderedList class, etc., to compile; we provide most of them here.

BuildBSTList.java is an (incomplete) program that starts with an ArrayOrderedList of integers from 1 to 10, and builds a BinarySearchTreeList by just taking the items from the ordered list (in order) and adding them to the binary search tree. What do you expect that the binary search tree will look like? This will be confirmed when you complete the program and run it; it will show the created tree in level-order. The program then builds another BinarySearchTreeList by calling a method buildBalanced that uses an idea for building a balanced tree: it creates an ordinary array that is the same size as the ArrayOrderedList, copies the items in ArrayOrderedList to this array, and then passes the array to a recursive method we have called buildBalancedRec here. The method buildBalanced is provided for you.

Run your completed program and note the difference in the level-order representation of the first tree created (by just adding the items in order) and the second tree created (by adding the items so that the tree is as balanced as possible). Draw a picture of the two trees from the level-order representation, and show it to the TA.
